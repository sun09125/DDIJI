전체 설계 명세서

OBI 띠지·표지 권위 온톨로지 — 데이터 명세서

한국 도서 띠지(belly-band)와 뒷표지(back cover) 의 마케팅 카피·권위·서지·인물·기관 정보를 RDF 로 표현하는 OWL 온톨로지의 공식 명세서.

본 명세서는 OBI 데이터를 활용하려는 외부 사용자가 자원의 구조·URI 패턴·SPARQL 쿼리 방법을 한 문서에서 파악할 수 있도록 작성되었다.

1. 소개

1.1 목적

띠지(belly-band)는 한국 출판 시장에서 가장 가시적인 마케팅 매체이다. 한 권의 책이 시점별로 다른 띠지를 두를 수 있으며, 그 위에 추천사·수상 정보·판매 실적·언론 평·시리즈 브랜딩 등 출판사의 마케팅 전략이 직접 드러난다. 그러나 띠지 텍스트는 ISBN 같은 표준 식별자가 없고, 출판사·시점·인물 관계가 복잡하게 얽혀 있어 구조화된 분석이 어려웠다.

OBI 띠지 온톨로지 는 이 텍스트들을 RDF 트리플로 보존·분석하기 위한 어휘 체계이다. 한 작품의 여러 판본, 한 판본의 여러 시점 띠지, 추천인·수상기관·시리즈 등 마케팅 카피 발화자를 모두 명시적으로 모델링한다.

1.2 범위

  • 대상 자료: 한국 도서 (소설 중심)
  • 수집 출처:
  • 알라딘 OpenAPI 메타데이터
  • 연도별 베스트셀러 (2000–2024)
  • 5개 주요 출판사 카테고리 페이지 (문학과지성사·문학동네·민음사·은행나무·창비)
  • 핵심 차별 기능: 한 판본의 여러 해 띠지 변화를 시점별 Image 인스턴스로 보존
  • 표지 권위 확장: 띠지뿐 아니라 뒷표지 마케팅·권위 텍스트까지 동일 어휘로 모델링 → "띠지(소모성·시장/수상 선언) vs 뒷표지(영구성·비평 논증)"의 표면별 권위 분업 분석 가능 (§14)

1.3 발행처·라이센스·연락처

항목 값

기반 URI http://aks.ac.kr/ontologies/obi#

인용 (추후 결정 — DOI 부여 시 명시)

1.4 사용 어휘

OBI 는 기존 국제 표준 어휘를 적극 차용하여 자체 어휘 신설을 최소화한다. 사용된 외부 어휘는 다음과 같다.

  • DC Terms (dcterms:) — 저자·출판사·포함 관계 등 범용 서지 어휘
  • FOAF (foaf:) — 사람·기관·행위 주체
  • BIBO (bibo:) — 도서 식별자·편집자·번역자
  • schema.org (schema:) — 출판일·장르
  • SKOS (skos:) — 분류 체계 (Concept·prefLabel·altLabel)

자체 어휘 obi: 는 띠지 마케팅 카피에 특수한 부분(블록 11종·시점별 Image·외부 LOD 매핑 정책)에만 사용된다.

2. 데이터 모델 개념도

데이터 모델 전체 개념도는 본 페이지 상단 데이터 모델 개념도 절에 수록되어 있다. FRBR 4계층(작품→판본→촬영본) 위에 표지 텍스트 산물 obi:CoverCopy 가 띠지(obi:Obi, 소모성)·뒷표지(obi:BCover, 영구성)로 분기하며, 두 면이 11종 마케팅·권위 블록을 공유 상속한다.

OBI 데이터를 SPARQL 로 조회하거나 RDF 도구에서 직접 다룰 때 사용하는 namespace prefix 일람.

4. 자원 식별 체계 (URI 패턴)

OBI 의 자원 URI 는 불투명(opaque) 식별자로 부여된다. URI 자체에는 이름·의미를 담지 않으며, 자원의 이름·제목 등 의미 정보는 rdfs:label·skos:prefLabel 에 둔다(이유 §10.2). 판본·촬영본·표지 산물은 ISBN13(+시점 슬롯)으로, 작품·인물·기관·분류 Concept 은 무의미 일련번호로 minting 한다.

자원 유형 URI 패턴 예시
작품 obi:Work_{nnnnnn} obi:Work_000001
시리즈 obi:Series_{nnnnnn} obi:Series_000078
판본 obi:Mani_{ISBN13} obi:Mani_9788936434120
촬영본 obi:Image_{ISBN}[_{year\|pub}] obi:Image_9788936434120_2021
띠지 obi:Obi_{ISBN}[_{year\|pub}] obi:Obi_9788936434120_2021
뒷표지 obi:BCover_{ISBN}[_{year\|pub}] obi:BCover_9788901105574
사람 obi:Person_{nnnnnn} obi:Person_000016
기관 obi:Org_{nnnnnn} obi:Org_000001
익명 독자 obi:AnonymousReader_{nnnnnn} obi:AnonymousReader_000001 (단일 인스턴스)
추천인 직군 obi:Role_{nnnnnn} obi:Role_000020
영상화 유형 obi:AdaptType_{nnnnnn} obi:AdaptType_000001
시리즈 유형 obi:SeriesType_{nnnnnn} obi:SeriesType_000002
수상 상태 obi:AwardStatus_{nnnnnn} obi:AwardStatus_000001
기관 역할 obi:OrgRole_{nnnnnn} obi:OrgRole_000001

따라서 출판사 "문학과지성사"는 URI 가 아니라 불투명 URI(예 obi:Org_000123)에 skos:prefLabel "문학과지성사"@ko 로 표상된다. 이명 통합(§10.3·§10.4)은 여러 표기를 하나의 불투명 URI 로 모아 대표 이름을 prefLabel, 이명을 altLabel 로 붙이는 과정이다. (뒷표지 신규 인물·기관 124건도 불투명 ID 로 정규화 완료 — 전 자원이 불투명 URI 이며 이름은 라벨로 보존.)

시점 슬롯의 의미

  • _{year} (예: _2021) — 연도별 베스트셀러 표지에서 검출된 띠지. 같은 ISBN 의 책이 2021·2022·2023 베스트셀러였다면 시점별로 별도 Image 가 생성된다.
  • _pub — 출판사 카테고리 페이지에서 단일 시점 수집된 띠지. 연도 정보가 없는 경우 사용.

5. 클래스 정의

OBI 의 자원은 7개 핵심 클래스로 분류된다. FRBR(Functional Requirements for Bibliographic Records) 의 4계층 모델을 따르되, 띠지의 텍스트 산물을 별도 클래스로 분리하였다.

5.1 핵심 자원 클래스

FRBR 4계층 + 시리즈로 구성된다.

클래스 부모 클래스 설명 예시
obi:Work (없음) 추상적 저작물. ISBN 무관 한 작품의 정체성. "소년이 온다"
obi:Series obi:Work 여러 작품을 묶는 시리즈. "위픽 시리즈"
obi:Manifestation (없음) ISBN 1:1 의 특정 판본. "소년이 온다 (창비, 양장본 2014)"
obi:Image (없음) 한 시점의 표지+띠지 디지털 스캔. "9788936434120 의 2021 베스트셀러 페이지 표지"
obi:Obi (없음) 띠지 텍스트 산물. Image 에서 추출. (위 Image 에서 OCR+LLM 으로 추출된 띠지 텍스트)

5.2 행위 주체 (FOAF 차용 + 자체 확장)

저자·추천인·출판사 등 띠지에 등장하는 모든 인물·기관·집단의 상위 클래스 체계.

클래스 부모 설명
foaf:Agent (없음) 모든 행위 주체의 상위
foaf:Person foaf:Agent 자연인 (저자·추천인·평론가)
foaf:Organization foaf:Agent 기관 (출판사·언론사·시상기관·대학)
obi:AnonymousReader foaf:Agent 익명 독자 placeholder. 단일 인스턴스 obi:AnonymousReader_독자 로 모든 익명 추천을 통합 표상한다. 식별 가능한 한 사람의 entity 가 아니라 익명 집단을 한 데 묶는 자리표시자다.

5.3 분류 체계 (SKOS)

클래스 설명
skos:Concept 분류 항목 (예: "평론가"·"드라마")
skos:ConceptScheme 분류 체계 자체 (5종, §7.2 참조)

6. 속성 정의

OBI 가 사용하는 속성은 자원 간 관계를 표현하는 Object Property 와 텍스트·날짜·식별자 등 값을 표현하는 Datatype Property 로 나뉜다.

6.1 Object Properties

6.1.1 FRBR 계층 체인

작품에서 띠지까지 4계층을 연결하는 핵심 관계.

속성 도메인 → 레인지 설명
obi:hasManifestation obi:Work → obi:Manifestation 한 작품이 여러 판본을 가질 수 있다
obi:hasImage obi:Manifestation → obi:Image 한 판본이 시점별 여러 촬영본을 가질 수 있다
obi:hasObi obi:Image → obi:Obi 한 촬영본에서 추출된 띠지
obi:extractedFrom obi:Obi → obi:Image hasObi 의 역방향

6.1.2 작품의 기여자

저자 외에 편집자·번역자·원작자·기타 기여자를 명시적으로 구분한다.

속성 도메인 → 레인지 설명
dcterms:creator obi:Work → foaf:Person 주 저자 (다중 가능)
bibo:editor obi:Work → foaf:Person 편집자 (선집 엮은이)
bibo:translator obi:Work → foaf:Person 번역자
obi:originalAuthor obi:Work → foaf:Person 원작자 (노벨라이즈·각색의 source)
dcterms:contributor obi:Work → foaf:Person 기타 기여자 (해설·사진작가)

6.1.3 출판사·시리즈

속성 도메인 → 레인지 설명
dcterms:publisher obi:Manifestation → foaf:Organization 현 판본 출판사
obi:formerPublisher obi:Manifestation → foaf:Organization 이전 출판사 (출판권 이양 이력)
dcterms:hasPart obi:Work → obi:Work 작품/시리즈가 다른 작품을 포함
obi:seriesType obi:Series → skos:Concept 시리즈 유형 (저자연작·큐레이션·전집)
obi:curator obi:Series → foaf:Agent 시리즈 기획자 (저자와 분리)
obi:organizationRole foaf:Organization → skos:Concept 기관의 다중 역할 태그

6.1.4 마케팅 카피 부착 (Obi → bnode)

띠지의 마케팅 카피는 11종 sub-property 로 분류되며 모두 익명 노드(bnode)에 연결된다.

속성 설명
obi:hasMarketingCopy 모든 카피 블록의 상위 속성
obi:hasPublisherCopy ① 출판사 카피
obi:hasEndorsement ② 추천사
obi:hasPressCitation ③ 언론 평
obi:hasMediaMention ④ 매체 언급
obi:hasAwardMention ⑤ 수상
obi:hasSelectionMention ⑥ 선정
obi:hasSalesRecord ⑦ 판매 실적
obi:hasAdaptationMention ⑧ 영상화
obi:hasSeriesBranding ⑨ 시리즈 표기
obi:hasAuthorNote ⑩ 작가의 말
obi:hasTextExcerpt ⑪ 본문 발췌

6.1.5 카피 블록 내부 관계

각 카피 블록(bnode) 안의 발화자·직군·소속·유형 등을 표현한다.

속성 도메인 → 레인지 설명
obi:issuedBy bnode → foaf:Agent 발화 주체 (출판사·언론사·시상기관)
obi:endorsedBy bnode → foaf:Agent 추천인
obi:endorserRole bnode → skos:Concept 추천인 직군 (배열 허용)
obi:endorserAffiliation bnode → foaf:Organization 추천인 소속
obi:authoredBy bnode → foaf:Person 작가의 말 작성자
obi:aggregatedBy bnode → foaf:Organization 판매 실적 집계 기관
obi:adaptationType bnode → skos:Concept 영상화 유형
obi:awardStatus bnode → skos:Concept 수상 상태
obi:refersToSeries bnode → obi:Series 참조 시리즈

obi:contentText string@ko 필수 — 띠지에 인쇄된 텍스트
obi:sourceText string@ko 외부 원본 출처 (작품명·인터뷰·신문명 등)
obi:awardName string@ko 상명 (예: "노벨문학상")
obi:awardYear gYear 수상 연도
obi:awardEdition positiveInteger 수상 회차 (예: 39)
obi:selectionCategory string@ko 선정 카테고리
obi:selectionYear gYear 선정 연도
obi:selectionScope string@ko 선정 범위
obi:salesMilestone string/integer 판매 실적
obi:citationYear gYear 언론 평·매체 언급 시점

6.2.4 외부 링크 provenance·신뢰도

각 foaf:Person·foaf:Organization 의 외부 LOD 링크 상태·근거·신뢰도를 자기기록(self-documenting)하는 주석 속성. 상세 의미는 §8.2 참조.

속성 도메인 xsd 형 설명
obi:linkageStatus foaf:Person/Organization string 링크 상태 8종 (linked_nlk · linked_external · not_in_nlk · ambiguous_homonym · foreign_nonkorean · corporate_body · collective)
obi:linkageMethod foaf:Person/Organization string 매칭 방법(다중 가능). 예: local_kac_kmo · aladin_bio · bio_worktitle · manual_verified
obi:linkageConfidence foaf:Person/Organization string 신뢰도 3등급 (high · medium · review)
obi:linkageEvidence foaf:Person/Organization integer 동명이인 NLK 후보 수 (미링크 분류의 증거)

7. 마케팅 카피 블록 11종 상세

띠지에 인쇄되는 마케팅 텍스트는 의미에 따라 11종 블록으로 분류된다. 각 블록은 공통 필드(contentText)와 블록별 고유 필드(추천사라면 endorsedBy·endorserRole, 수상이라면 awardName·awardStatus 등)를 갖는다.

7.1 블록별 필드 사용 매트릭스

각 블록 유형에서 어떤 필드를 사용하는지 정리한다. ✓ = 필수, ◐ = 선택.

블록 content source issuedBy endorsedBy endorserRole endorserAffil authoredBy aggregated award* selection* sales adaptation refers citation
① 출판사 카피 ✓ ◐ ◐
② 추천사 ✓ ◐ ◐ ◐ ◐
③ 언론 평 ✓ ◐ ◐ ◐
④ 매체 언급 ✓ ◐ ◐ ◐
⑤ 수상 ✓ ◐ ◐ ✓
⑥ 선정 ✓ ◐ ◐ ✓
⑦ 판매 실적 ✓ ◐ ✓
⑧ 영상화 ✓ ◐ ◐ ✓
⑨ 시리즈 표기 ✓ ✓
⑩ 작가의 말 ✓ ◐ ◐
⑪ 본문 발췌 ✓ ◐

7.2 분류 체계 5종 (SKOS Concept Schemes)

마케팅 블록에서 발화자의 직군·영상화 유형·시리즈 유형 등을 표현하기 위한 표준 분류 체계 5종.

EndorserRoleScheme (추천인 직군 분류) 추천 블록의 endorserRole 값. 다중 직위 부착 가능 (예: 평론가 + 교수). 예시: obi:Role_소설가, obi:Role_평론가, obi:Role_교수, obi:Role_배우, obi:Role_시인, obi:Role_에세이스트, obi:Role_기자

AdaptationTypeScheme (영상화 유형 분류) hasAdaptationMention 블록의 adaptationType 값. 예시: obi:AdaptType_영화, obi:AdaptType_드라마, obi:AdaptType_뮤지컬, obi:AdaptType_오디오북

SeriesTypeScheme (시리즈 유형 분류) obi:seriesType 값. 3종.

Concept 설명 예
obi:SeriesType_저자연작 한 저자의 연작 해리포터
obi:SeriesType_큐레이션시리즈 출판사 기획 시리즈 위픽 시리즈, 문학동네 시인선
obi:SeriesType_전집 번역·고전 중심 완결 세트 열린책들 세계문학전집

AwardStatusScheme (수상 상태 분류) 3종: obi:AwardStatus_수상, obi:AwardStatus_최종후보, obi:AwardStatus_후보

OrganizationRoleScheme (기관 역할 분류) 한 기관 다중 역할 가능 (예: 창비 = 출판사 + 수상주체). 예시: obi:OrgRole_출판사, obi:OrgRole_언론, obi:OrgRole_수상주체, obi:OrgRole_집계기관, obi:OrgRole_기획레이블, obi:OrgRole_제작사

8. 외부 LOD 매핑 정책

OBI 의 인물·기관·작품·판본은 가능한 한 외부 LOD 자원에 연결된다. 매핑 술어는 자원의 의미적 동등성에 따라 신중히 선택된다.

8.1 매핑 술어

대상 클래스 매핑 술어 외부 LOD
foaf:Person owl:sameAs NLK KAC (1차 매핑). KAC 자체가 Wikidata · VIAF · ISNI · GND · BNF · BNE · NDL · LoC · DBpedia · Libris · WorldCat · IDref 와 매핑되어 있어, 한 작가에 대해 10여 개 외부 자원이 자동 연결된다.
foaf:Organization owl:sameAs NLK KAB · Wikidata · VIAF · ISNI
obi:Work owl:sameAs Wikidata Q-id (NLK 측 Work 단위 자원 없음)
obi:Manifestation skos:closeMatch NLK KMO. owl:sameAs 대신 closeMatch 를 쓰는 이유: KMO 는 MARC 서지 record 로 한 KMO 가 여러 ISBN 을 묶을 수 있어 카디널리티가 1:1 이 아니다. sameAs 적용 시 reasoner 가 두 Manifestation 을 잘못 동일시할 위험이 있다.
주제 dcterms:subject NLK KSH URI 직접 값
장르 schema:genre Wikidata Q-id 직접 값

기관 간 계층은 모델링하지 않는다. 모회사·자회사·imprint 관계는 한국 출판업계 자료가 부족하고 시기별 변동이 심해, 외부 권위 데이터(Wikidata·VIAF)에 위임한다.

8.2 링크 상태·신뢰도 (provenance 주석)

모든 foaf:Person·foaf:Organization 은 외부 링크 상태를 obi:linkageStatus 로 자기기록한다. 설계 원칙은 "링크 가능한 건 전부 링크, 불가능한 건 전부 사유 범주화" — 미분류 인스턴스는 0건이다. 이로써 사용자는 "왜 이 인물은 KAC 가 없는가"를 추측하지 않고 그래프에서 직접 읽을 수 있다.

링크 상태 8종 (obi:linkageStatus)

값 의미
linked_nlk NLK 권위(KAC/KAB) 직결
linked_external NLK 없음, 알라딘/Wikidata/VIAF/ISNI 등 외부만 보유
ambiguous_homonym 미링크 + NLK 동명이인 후보 2+ (수동 영역). 후보 수는 obi:linkageEvidence
not_in_nlk 미링크 + NLK 미수록 (외국 표기·필명 포함)
foreign_nonkorean 미링크 + 비한글 표기
corporate_body 단체·법인-저자
collective 복수 인명 결합 문자열

신뢰도 3등급 (obi:linkageConfidence) — KAC 매칭의 근거 강도

등급 의미 주요 방법
high 작품(KMO)겹침·생년·대표작·사람검증 등 강증거 local_kac_kmo · aladin_bio · bio_worktitle · manual_verified
medium 단일 비문학 후보·직업 유일생존 등 보조 증거 local_kac_single · coauthor · kn_*
review 저정밀 자동매칭·모호 — 수동 검토 권장 kmo_title · local_kac_ambiguous

obi:linkageMethod 는 매칭이 어느 경로로 이뤄졌는지 기록하며(다중 가능), 검수 우선순위 판단·재현에 쓰인다.

8.3 동명이인 disambiguation 방법론

동명이인 해소는 다음 신호를 증거 강도 순으로 결합한다. 모두 사람의 주관이 아닌 데이터 증거에 기반한다.

  1. KMO 작품 겹침 — NLK 후보의 등록작(KMO)이 ABox 판본의 KMO 와 일치 (최강)
  2. 생년 일치 — 알라딘 작가 bio 의 출생연도 ↔ NLK birthYear
  3. 대표작 제목 일치 — bio 의 대표작(《》) ↔ 후보의 등록작 제목(book.json title)
  4. 직업 일치 — bio 의 직업 ↔ NLK fieldOfActivity/jobTitle. 영화감독·가수·사진가 등 비문학 창작자(추천·원작 역할)도 포함

알라딘 bio 활용: 알라딘 authors[].authorInfo(작가 소개)에 담긴 생년·직업·대표작은 동명이인 해소의 핵심 단서다. 유명도 추정 같은 휴리스틱보다 안전하며, 잘못 붙은 링크를 자동 교정하는 데에도 쓰였다.

사람 검수 루프: 자동 매칭 결과는 인터랙티브 HTML 검수 도구로 사람이 승인/반려/수정할 수 있으며, 확정분은 manual_verified(high)로 기록되고 잘못된 링크는 제거된다.

10. 명명 규약

표기 규약: 본 절에서 Org_문학과지성사·Org_RHK 처럼 적은 것은 가독성을 위한 대표 라벨(prefLabel) 지칭이며, 실제 URI 는 불투명 일련번호(obi:Org_000123)다(§4).

10.1 정규화 키(slug) 생성 룰

slug 는 URI 가 아니라 이명 통합용 정규화 키(대표 prefLabel 산출용)다. 자원 이름에서 한글·영문·숫자만 남기고 공백·특수문자를 제거해, 같은 entity 의 표기 변이를 한 키로 모은다. 실제 URI 는 이 키와 무관하게 불투명 일련번호(§4)로 부여된다.

def slugify(name: str) -> str:
    return re.sub(r"[^가-힣A-Za-z0-9]", "", name or "")

예: "문학과 지성사"·"문학과의 지성사" → 정규화 키 문학과지성사 → 한 entity(obi:Org_000123, prefLabel "문학과지성사")로 통합.

10.2 라벨 부착 정책

한 자원에 같은 언어의 라벨이 여럿 있을 때 rdfs:label 을 중복 선언하지 않는다. 대표 이름만 rdfs:label + skos:prefLabel, 이명은 skos:altLabel 로 분리한다.

예: obi:Org_창비 → prefLabel "창비"@ko, altLabel "창작과비평사"@ko

10.3 출판사 정규화

메타 publisher 문자열의 X(Y) 패턴 (예: "알에이치코리아(RHK)") 은 4 유형으로 분류된다.

유형 정책 예
Type a — 풀네임/약자 같은 entity 출판업계 통용 표기를 slug + 다른 표기를 skos:altLabel Org_RHK + altLabel "알에이치코리아"
Type b — 표기 변이 같은 entity, altLabel 부착 Org_한즈미디어 + altLabel "한스미디어"
Type c — imprint/모회사 X 만 slug, Y 는 rdfs:comment 로 보존 Org_퍼플레인 + comment "메타데이터 원문: 퍼플레인(갈매나무)"
Type d — 법인 접미 X 만 사용, "(주)" 무시 Org_데이즈엔터

10.4 OCR 변이·표기 변이 자동 통합

LLM 이 추출한 OCR 오인식 표기는 사전 정의된 매핑(publisher_normalizer.py 의 _OCR_VARIANT_MAP·_AWARD_BODY_MAP)으로 canonical slug 에 자동 매핑된다.

예: - "문학과의 지성사" → Org_문학과지성사 - "엘리시트" / "엘리시온" → Org_엘릭시르 - "서울대학교" → Org_서울대 - "한겨레신문사" → Org_한겨레 - "부커상 위원회" → Org_부커상재단

11. SPARQL 사용 예시

외부 사용자가 자주 쓸 만한 분석 패턴 5종.

11.1 한 작가의 모든 띠지 시점

PREFIX obi: <http://aks.ac.kr/ontologies/obi#>
PREFIX dcterms: <http://purl.org/dc/terms/>
PREFIX bibo: <http://purl.org/ontology/bibo/>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

SELECT ?work_label ?isbn ?year WHERE {
  obi:Person_한강 ^dcterms:creator ?work .
  ?work rdfs:label ?work_label ;
        obi:hasManifestation ?mani .
  ?mani bibo:isbn13 ?isbn ;
        obi:hasImage ?img .
  ?img obi:hasObi/obi:collectionDate ?year .
}
ORDER BY ?work_label ?year

11.2 영상화된 작품의 추천인 직군 분포

PREFIX obi: <http://aks.ac.kr/ontologies/obi#>
PREFIX skos: <http://www.w3.org/2004/02/skos/core#>

SELECT ?role (COUNT(*) AS ?n) WHERE {
  ?work obi:hasManifestation/obi:hasImage/obi:hasObi ?obi .
  ?obi obi:hasAdaptationMention [] .
  ?obi obi:hasEndorsement/obi:endorserRole/skos:prefLabel ?role .
}
GROUP BY ?role ORDER BY DESC(?n)

11.3 출판사별 추천인 직군 다양성

PREFIX obi: <http://aks.ac.kr/ontologies/obi#>
PREFIX dcterms: <http://purl.org/dc/terms/>
PREFIX skos: <http://www.w3.org/2004/02/skos/core#>
PREFIX foaf: <http://xmlns.com/foaf/0.1/>

SELECT ?publisher (COUNT(DISTINCT ?role) AS ?role_diversity) WHERE {
  ?mani dcterms:publisher/foaf:name ?publisher ;
        obi:hasImage/obi:hasObi/obi:hasEndorsement/obi:endorserRole/skos:prefLabel ?role .
}
GROUP BY ?publisher
ORDER BY DESC(?role_diversity)
LIMIT 20

11.4 시리즈별 마케팅 카피 패턴

PREFIX obi: <http://aks.ac.kr/ontologies/obi#>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>
PREFIX dcterms: <http://purl.org/dc/terms/>

SELECT ?block_type (COUNT(*) AS ?n) WHERE {
  obi:Series_위픽시리즈 dcterms:hasPart ?work .
  ?work obi:hasManifestation/obi:hasImage/obi:hasObi ?obi .
  ?obi ?p ?b .
  ?p rdfs:subPropertyOf+ obi:hasMarketingCopy .
  BIND(REPLACE(STR(?p), "^.*#", "") AS ?block_type)
}
GROUP BY ?block_type ORDER BY DESC(?n)

11.5 다년도 띠지 변화 추적

같은 ISBN 의 책이 여러 해에 걸쳐 다른 띠지를 두른 경우를 식별한다.

PREFIX obi: <http://aks.ac.kr/ontologies/obi#>
PREFIX bibo: <http://purl.org/ontology/bibo/>

SELECT ?isbn (COUNT(DISTINCT ?obi) AS ?n_obi) WHERE {
  ?mani bibo:isbn13 ?isbn ;
        obi:hasImage/obi:hasObi ?obi .
}
GROUP BY ?isbn
HAVING (COUNT(DISTINCT ?obi) > 1)
ORDER BY DESC(?n_obi)

11.6 신뢰도 높은 인물 링크만 선별

분석·시각화 시 저신뢰(review) 노드를 제외하고 검증된 링크만 사용한다.

PREFIX obi: <http://aks.ac.kr/ontologies/obi#>
PREFIX owl: <http://www.w3.org/2002/07/owl#>
PREFIX foaf: <http://xmlns.com/foaf/0.1/>

SELECT ?person ?name ?kac WHERE {
  ?person a foaf:Person ;
          obi:linkageConfidence "high" ;
          foaf:name ?name ;
          owl:sameAs ?kac .
  FILTER(CONTAINS(STR(?kac), "/resource/KAC"))
}

미검수 대상(수동 검토 큐)만 뽑으려면 obi:linkageConfidence "review" 로 바꾼다.

12. 데이터 통계 (2026-06-23 기준, rdflib 7.6 파싱 검증)

본 절 수치는 최종본 .ttl/.owl 파일을 rdflib 로 직접 파싱해 재계산했다. 5개 파일 전부 파싱 정상(구문 오류 0).

12.1 인스턴스 수 (통합그래프)

클래스 수
obi:Work 2,211
obi:Manifestation 2,382
obi:Image 2,933
obi:Obi 2,935
obi:Series 103
foaf:Person 2,505
foaf:Organization 783
skos:Concept 144
skos:ConceptScheme 5
obi:AnonymousReader 1

obi:AnonymousReader 1건(obi:AnonymousReader_000001)은 익명 독자·리뷰어 발화(알라딘 마스킹 닉네임 "bri"·"무지개님" 등)를 통합 표상하는 placeholder 인스턴스다. obi:AnonymousReader rdfs:subClassOf foaf:Agent 로 정의되어 *의도적으로 foaf:Person 이 아니다(식별 가능한 한 사람이 아니라 익명 집단이므로). endorsedBy 대상 중 유일하게 foaf:Person 이 아닌 노드이며, 닉네임 원문은 각 추천 블록의 rdfs:comment 에 보존된다. 데이터 오류가 아니라 설계상 정상이다.

12.2 외부 링크 현황

발화주체 342건 중 1건은 일반 추천인이 아니라 익명 독자 placeholder obi:AnonymousReader_000001(§12.1)로, 통합그래프에 a obi:AnonymousReader(⊂ foaf:Agent)로 정상 타이핑된 기존 공유 인스턴스다. 따라서 실제 신규 인물 은 125명이며 모두 foaf:Person 으로 타이핑돼 있다(미타이핑 0 — 데이터 오류 없음).

표면별 권위 분업 발견(분석 출처: DH2026/11_표면별권위분업/): 비평(평론가) 권위 블록은 띠지만 보면 82건이나 뒷표지 포함 시 320건으로 늘어, 평론가 권위 비중이 11%→21%(약 4배)로 재평가된다. 뒷표지 총 권위(759)는 띠지(752)와 사실상 동등하다. 즉 띠지=동료추천+수상(선언), 뒷표지=동료+비평(논증)으로 권위 양식이 분업된다. (이 분업 수치는 역할 필터 기반 분석 산출물로, 위 블록 총량 통계와 집계 단위가 다름.)

14. 표지 권위 확장 (Cover-Authority Extension)

이 온톨로지는 띠지 전용 모델을 표지 전체 권위 로 확장한다. 확장 어휘(CoverCopy·BCover·SurfaceScheme·printedOn·isEphemeral)는 제출 스키마 OBI_온톨로지_스키마.owl 에 통합되어 있다. 핵심만 요약한다.

14.1 상속 뿌리 obi:CoverCopy

obi:CoverCopy (표지 텍스트 산물)            ← 신설 상위(상속 뿌리)
 ├ obi:Obi    (띠지,   isEphemeral=true)    ← 소모성·즉각 마케팅
 └ obi:BCover (뒷표지, isEphemeral=false)   ← 영구·비평적 정당화

11종 마케팅 블록 속성(hasEndorsement·hasPressCitation 등)의 domain 을 obi:CoverCopy 에 둠으로써 띠지·뒷표지가 동일 어휘를 공유 상속한다. 인물(foaf:Person)·작품(obi:Work)·기관(foaf:Organization)은 그대로 재사용하여 기존 KAC/bio 매칭·신뢰도 인프라를 뒷표지 추천인에도 동일 적용한다.

14.2 위치(surface) 2계층 · 물질성

  • 거친 위치(띠지 vs 뒷표지) = 캐리어 클래스(Obi/BCover)가 인코딩.
  • 미세 위치(앞면/뒷면/날개) = obi:printedOn → obi:SurfaceScheme(SKOS) Concept. 클래스 계층은 불변, 값만 분류.
  • 물질성 = obi:isEphemeral (소모성/영구성)을 캐리어 클래스에 인코딩 → "교체되는 마케팅 면 vs 영구 정당화 면" 분석.

14.3 무(無)reasoner 주의 (materialize)

OBI 도구는 reasoner 없이 rdf:type 만 직접 본다. 상속을 데이터에 실체화하려면 BCover/Obi 인스턴스를 obi:CoverCopy 로도 함께 타이핑한다(뒷표지 ABox 는 이미 적용됨 — ?x a obi:CoverCopy 즉시 작동). 상속 속성 경로 질의는 rdfs:subPropertyOf* 를 명시한다(OBI_SPARQL질의.rq 07번 참조).
